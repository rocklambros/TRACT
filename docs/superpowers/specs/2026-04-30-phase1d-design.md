**PRD Sections:** 6.9 (CLI Tool), 6.10 (Hub Proposal System)
**Date:** 2026-04-30
**Status:** Approved after 2-cycle adversarial review (12 agents, 7 rounds)

---

## 1. Goals

Build a production CLI (`tract`) with 8 commands that expose Phase 1B/1C's trained model, calibration, and crosswalk database to end users. Build a guardrailed hub proposal system for extending the CRE ontology when OOD controls are detected.

**Success criteria:**
- All 8 CLI commands functional with `--help`, `--json`, and example-rich documentation
- `tract assign` returns results in < 30 seconds (cold start including model load on Jetson Orin AGX)
- Hub proposal system: end-to-end OOD detect → cluster → guardrails → propose → review
- All commands tested: golden path + edge cases (empty input, unknown framework, malformed JSON)
- No injection vectors in any user input path

## 2. Architecture

### 2.1 New Modules

```
tract/

```

### 2.2 Deployment Artifacts (generated by updated T5)

T5 gains a single consolidated NPZ file and a calibration bundle:

```
results/phase1c/deployment_model/
  model/                        # Existing deployment model weights
  deployment_artifacts.npz      # NEW — single file containing:
    hub_embeddings              # 522×1024 float32
    control_embeddings          # 2754×1024 float32
    hub_ids                     # sorted(hierarchy.hubs.keys()) — canonical ordering
    control_ids                 # control ID index matching embedding rows
    model_adapter_hash          # SHA-256 of adapter_model.safetensors
    generation_timestamp        # ISO 8601
  calibration.json              # NEW — bundled calibration parameters:
    t_deploy                    # Temperature scaling parameter (0.074)
    ood_threshold               # Max-cosine OOD threshold (0.568)
    conformal_quantile          # Conformal prediction quantile (0.997)
    global_threshold            # Binary assignment threshold (0.121)
    hierarchy_hash              # SHA-256 of cre_hierarchy.json at generation time
    calibration_note            # "Calibrated on 420 traditional framework holdout items.
                                #  Accuracy on AI framework text may differ."
```

**Design decision (from adversarial review):** Single NPZ file instead of separate hub_embeddings.npz + control_embeddings.npz. One file = one version check = no cross-file mismatch.

**Design decision (from adversarial review):** Hash `adapter_model.safetensors` (1.6MB LoRA adapter, SHA-256 in <10ms), not config files. Config hash doesn't detect weight changes from different training data.

### 2.3 Data Flow per Command

| Command | Input | Library calls | Output |
|---------|-------|---------------|--------|
| `assign` | Text string or `--file` | `inference.predict()` → calibrate → conformal → OOD | Top-K hubs + confidence + hierarchy paths |
| `compare` | 2+ framework IDs | `compare.cross_framework_matrix()` via crosswalk.db | Relationship matrix (equivalent/related/gap) |
| `ingest` | Framework JSON file | `schema.validate()` → `inference.predict_batch()` → duplicates | Review JSON with predicted assignments |
| `export` | Format + filters | `crosswalk.export.export_crosswalk()` (existing) | CSV/JSON/JSONL file |
| `hierarchy` | Hub ID | `CREHierarchy` + `crosswalk.store` queries | Path + description + siblings + linked controls |
| `propose-hubs` | (reads OOD from artifacts) | `proposals.cluster` → `proposals.guardrails` → `proposals.naming` | `hub_proposals/round_N/` directory |
| `review-proposals` | Round directory | `proposals.review` interactive loop | Updated hierarchy + crosswalk.db |
| `tutorial` | (none) | Runs live queries against model + DB | Guided walkthrough |

## 3. `tract/inference.py` — Model Loading & Prediction

### 3.1 TRACTPredictor

```python
class TRACTPredictor:
    """Loads deployment model + cached artifacts. Stateful — holds model in memory."""
    
    def __init__(self, model_dir: Path):
        # 1. Load SentenceTransformer model from model_dir / "model"
        # 2. Load deployment_artifacts.npz
        # 3. Verify model_adapter_hash matches current adapter_model.safetensors
        # 4. Verify hierarchy_hash from calibration.json matches current cre_hierarchy.json
        # 5. Load calibration.json (T_deploy, OOD threshold, conformal quantile)
        # 6. Health check: encode known calibration item, assert cosine to known hub > 0.3
        
    def predict(self, text: str, top_k: int = 5) -> list[HubPrediction]:
        """Single control text → top-K hub assignments with calibrated confidence."""
        # 1. sanitize_text(text) — defense in depth, caller never touches raw text
        # 2. model.encode(text) → 1×1024
        # 3. cosine against hub_embeddings → 522 scores (using canonical hub_ids ordering)
        # 4. softmax(similarities / T_deploy) → calibrated probabilities
        # 5. check OOD (max_sim < ood_threshold)
        # 6. build conformal prediction set (scores above conformal_quantile)
        # 7. return top-K with all fields
    
    def predict_batch(self, texts: list[str], top_k: int = 5) -> list[list[HubPrediction]]:
        """Batch prediction for tract ingest."""
        
    def find_duplicates(
        self, text: str, 
        duplicate_threshold: float = 0.95,
        similar_threshold: float = 0.85,
    ) -> tuple[list[DuplicateMatch], list[DuplicateMatch]]:
        """Compare text embedding against all control_embeddings.
        Returns (duplicates above 0.95, similar controls above 0.85)."""
```

### 3.2 Data Types

```python
@dataclass
class HubPrediction:
    hub_id: str
    hub_name: str
    hierarchy_path: str
    raw_similarity: float           # Cosine similarity (model's actual output)
    calibrated_confidence: float    # Temperature-scaled probability
    in_conformal_set: bool
    is_ood: bool

@dataclass  
class DuplicateMatch:
    control_id: str
    framework_id: str
    title: str
    similarity: float
    tier: str                       # "duplicate" (≥0.95) or "similar" (≥0.85)
```

### 3.4 Key Decisions

- **Calibrated probability is the PRIMARY displayed metric** (per CLAUDE.md: "Always calibrate before reporting confidence scores"). `--raw` flag shows cosine similarity. `--verbose` shows both. Calibration limitation documented in output: "(calibrated on traditional frameworks)".
- **Sanitization inside `predict()`** — caller never touches raw text.
- **OOD flag is informational** — `predict()` still returns hub assignments for OOD items.
- **Health check at init** — encode a known text, verify cosine to known hub exceeds floor (0.3). Catches corrupted model files.
- **Cold start is ~16 seconds on Jetson** (13s import + 2s load + 1s health check). Within 30s PRD budget. Documented in `--help`.

## 4. `tract/compare.py` — Cross-Framework Relationships

Factored from T5's `_export_cross_framework_matrix()` into a reusable library function.

```python
def cross_framework_matrix(
    db_path: Path,
    framework_ids: list[str],
    hierarchy: CREHierarchy,
) -> CrossFrameworkResult:
    """Live query: relationship matrix between 2+ frameworks.
    
    Equivalent: controls assigned to the same hub.
    Related: controls whose hubs share a parent (via hierarchy.get_parent()).
    Gap: controls with no cross-framework match.
    """

@dataclass
class CrossFrameworkResult:
    equivalences: list[Equivalence]
    related: list[RelatedPair]
    gap_controls: dict[str, list[str]]       # framework_id → unmatched control_ids
    framework_pair_overlap: dict[tuple[str, str], int]
    total_shared_hubs: int
```

- **"Related pairs" traversal uses `CREHierarchy.get_parent()`** — not reimplemented in compare.py. The hierarchy class owns traversal logic.
- **Filters to requested frameworks** — not all 31.
- **Returns structured dataclasses** — CLI formats as table or JSON.

## 5. `tract/proposals/` — Hub Proposal System

### 5.1 `cluster.py` — HDBSCAN Clustering

```python
def cluster_ood_controls(
    embeddings: np.ndarray,        # N × 1024, from deployment_artifacts.npz
    control_ids: list[str],
    min_cluster_size: int = 3,     # PHASE1D_HDBSCAN_MIN_CLUSTER_SIZE
    min_samples: int = 2,          # PHASE1D_HDBSCAN_MIN_SAMPLES
) -> list[Cluster]:
    """HDBSCAN clustering on OOD control embeddings.
    
    Uses euclidean metric on L2-normalized vectors (equivalent to cosine distance).
    Deterministic: same inputs → same clusters. Verified on Jetson ARM64.
    
    Returns empty list if insufficient OOD items — this is expected behavior.
    CLI output: "No proposals generated (insufficient OOD controls)."
    """

@dataclass
class Cluster:
    cluster_id: int
    control_ids: list[str]
    centroid: np.ndarray
    nearest_hub_id: str
    nearest_hub_similarity: float
    member_frameworks: set[str]
```

**OOD source:** `propose-hubs` loads `deployment_artifacts.npz`, recomputes OOD flags for ALL controls in the NPZ (all frameworks in crosswalk.db) by comparing cached control embeddings against hub embeddings using the OOD threshold from `calibration.json`. Does NOT read stale `is_ood` flags from crosswalk.db assignments. Does NOT require model loading. OOD detection is framework-agnostic — any control with max cosine < threshold is a candidate.

**Curse of dimensionality:** HDBSCAN in 1024 dims with <100 items may find zero clusters. This is CORRECT behavior — it means there isn't enough OOD signal to propose hubs. The pipeline is a latent capability that activates after `tract ingest` brings frameworks with unmapped concepts. Documented in CLI output and README.

### 5.2 `guardrails.py` — 6-Filter Pipeline

```python
def apply_guardrails(
    clusters: list[Cluster],
    hierarchy: CREHierarchy,
    hub_embeddings: np.ndarray,
    hub_ids: list[str],
    control_metadata: dict[str, dict],
    budget_cap: int = 40,
) -> list[GuardrailResult]:
    """Apply 6 guardrails to each cluster.
    
    Guardrails:
    1. Minimum evidence: 3+ controls from 2+ frameworks
    2. Hierarchy constraint: parent hub identified via max cosine to existing hubs.
       Flag cosine < 0.20 as "uncertain placement" in output.
    3. Inter-cluster separation: proposed cluster centroids must have pairwise
       cosine < 0.7 to each other. Prevents redundant proposals.
       (NOT cluster-to-hub separation — that's always satisfied for OOD items.)
    4. Budget cap: top 40 by evidence strength if more pass filters 1-3
    5. Candidate queue: write proposals with evidence to hub_proposals/round_N/
    6. Determinism: verified by test fixture on deployment platform
    """
```

**Design decision (from adversarial review):** Original guardrail 3 (similarity ceiling < 0.85 to existing hubs) was dead code — OOD items by definition have max cosine < 0.568 to hubs, so no centroid could reach 0.85. First proposed replacement (centroid > OOD threshold distant from nearest hub) was also dead — same logic. Correct guardrail is **inter-cluster separation**: prevents proposing two clusters that should be one.

## 7. `tract ingest` — Framework Ingestion Pipeline

### 7.1 Pipeline Steps

```
Input JSON file
    │
    ▼
1. File size check (reject > 50MB — PHASE1D_INGEST_MAX_FILE_SIZE)
    │
    ▼
2. Schema validation (tract/schema.py FrameworkOutput Pydantic model)
    │ Fail → error with specific validation messages
    ▼
3. Framework ID collision check (already in crosswalk.db?)
    │ Collision → error unless --force
    ▼
4. Sanitize all control texts (sanitize_text on title + description)
    │
    ▼
5. Batch inference (TRACTPredictor.predict_batch)
    │ → top-K hub assignments per control, calibrated, with OOD flags
    ▼
6. Exhaustive duplicate detection (TRACTPredictor.find_duplicates per control)
    │ → "duplicate" tier at cosine ≥ 0.95, "similar" tier at cosine ≥ 0.85
    ▼
7. Write review file (atomic write to output path)
```

### 7.2 Review File Format

```json
{
  "framework_id": "new_framework_v1",
  "framework_name": "New Framework",
  "generated_at": "2026-04-30T12:00:00Z",
  "model_version": "deployment_round2",
  "context": "ingestion",
  "summary": {
    "total_controls": 45,
    "ood_flagged": 3,
    "duplicate_flagged": 2,
    "similar_flagged": 5,
    "high_confidence": 28,
    "low_confidence": 14
  },
  "controls": [
    {
      "control_id": "NF-01",
      "title": "Access Control Policy",
      "predictions": [
        {
          "hub_id": "646-285",
          "hub_name": "...",
          "raw_similarity": 0.52,
          "confidence": 0.82,
          "in_conformal_set": true
        }
      ],
      "is_ood": false,
      "duplicates": [
        {"control_id": "csa_aicm::AIC-01", "similarity": 0.97, "title": "...", "tier": "duplicate"}
      ],
      "similar": [
        {"control_id": "mitre_atlas::AML.T0001", "similarity": 0.88, "title": "...", "tier": "similar"}
      ]
    }
  ]
}
```

**Design decision (from adversarial review):** Review files store BOTH `raw_similarity` and `confidence`. The `context` field ("active_learning" or "ingestion") disambiguates `is_ood` semantics: in AL context, OOD means "relative to training distribution"; in ingestion context, OOD means "relative to all known hubs."

### 7.3 No Auto-Insert

`tract ingest` generates a review file, it does NOT modify crosswalk.db. Expert must review first. Accepted results are inserted via existing `insert_assignments()`.

## 10. Adversarial Review Summary

Two review cycles, 12 agents, 7 rounds. Key findings and resolutions:

### Critical Findings Resolved

| Finding | Resolution |
|---------|------------|
| Calibration fit on wrong distribution (traditional vs AI) | Document limitation. Show calibrated as primary (per CLAUDE.md), `--raw` for cosine. |
| Original guardrail 3 dead code (similarity ceiling never fires on OOD items) | Replaced with inter-cluster separation (pairwise centroid cosine < 0.7) |
| First replacement guardrail ALSO dead (same OOD logic trap) | Corrected to inter-cluster check (operates within proposal set, not against existing hubs) |
| Hub proposal pipeline has 5 OOD items (zero clusters expected) | Latent capability. Activates after `tract ingest`. Tested with real held-out framework. |
| No model-embedding version binding | SHA-256 of adapter_model.safetensors stored in NPZ, verified at load |
| Config hash doesn't detect weight changes | Hash weights file, not config. 1.6MB adapter, <10ms. |
| Hub embedding column ordering could mismatch calibration | Canonical `sorted(hub_ids)` stored in NPZ, asserted at load. Existing code already uses this ordering. |

### Design Decisions from Review

| Decision | Rationale |
|----------|-----------|
| Single NPZ for all deployment artifacts | One file = one version check. No cross-file mismatch. |
| Calibrated probability as primary metric | CLAUDE.md mandates calibration. `--raw` for experts who want cosine. |
| Review files store both raw_similarity and confidence | Prevents representation split between CLI and review files. |
| "Hub similarity" vs "duplicate similarity" labels | Different cosine spaces (0.3-0.7 vs 0.85-0.99) need different labels. |
| Real held-out framework for proposal testing | Synthetic embeddings prove code runs, not algorithmic correctness. |
| T=0.074 extreme sharpening acknowledged | Top-1 dominates probability mass. Documented. Future work: explore isotonic regression. |

### Deferred Items

| Item | Phase |
|------|-------|
| Recalibrate on AI framework data | Requires labeled AI holdout (not available) |
| Cross-platform HDBSCAN determinism | Phase 2 (multi-platform deployment) |
| Isotonic regression as alternative calibration | Phase 2 (with more labeled data) |
| Recalibrate after hub insertion | Only needed if hub count changes >5% |

## 12. Security

- All user text input sanitized via `sanitize_text()` (strip null bytes, NFC normalize, 2000-char max) inside `predict()`.
- No `eval()`, `exec()`, `subprocess.shell=True`, or `pickle`.
- File size limit on `tract ingest` (50MB).
- `--name-with-llm` is opt-in with explicit help text: "Sends control texts to Claude API."
- Framework ID validated against `PHASE1A_FRAMEWORK_SLUG_RE` pattern.
- SQLite queries use parameterized statements, never string interpolation.
- All file writes use atomic patterns (temp + `os.replace()`).
